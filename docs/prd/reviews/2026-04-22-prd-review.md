# Orpheus PRD 审阅报告

> 状态：初版审阅结论，尚未合入 PRD
> 审阅日期：2026-04-22
> 审阅对象：`docs/prd/prd.md` v0.34（2488 行）+ `docs/prd/source-of-truth-index.md` + `docs/prd/agent-conventions.md` + `docs/prd/specs/` 8 份文件
> 审阅基准：前一轮市场调研结论（Inworld/Kokoro 双轨、自部署陷阱、Raphael 分发模式）+ MVP 通用规范
> 评级口径:🔴 阻塞性问题 / 🟡 重要建议 / 🟢 优化建议

## 一、总体印象

PRD 在**"产品深度"**与**"文档治理"**两个维度做得罕见地认真：source-of-truth-index + agent-conventions + 8 份 specs 的真源拆分，已经接近企业级规范。**真正的问题不是 PRD 写得不好，而是 PRD 的"产品骨架"和已经摸清的"市场打法"严重错位** —— PRD 还在写一个完整 SaaS 平台，而调研结论指向的是一个"Raphael 式的工具站 + 极窄 niche 切入"。

---

## 二、🔴 阻塞性问题（必须先解决再开工）

### R1. V1 范围与已确立的市场打法严重脱节

**位置**：`prd.md:627-643`（V1 范围 13 项）+ `prd.md:688-700`（9 种界面语言）+ `prd.md:709-716`（4 种音色语言）+ `prd.md:830-837`（首批 8 个旗舰音色）

调研已结论：**第一刀只切英语 × 悬疑/黑色叙事**。但 PRD 仍把 4 niche × 4 生成语言 × 9 UI 语言 × 8 旗舰音色 + 私有克隆 + 三层音色库 + 运营后台 9 视图全部压进 V1。这套范围至少是"V2-V4 路线图"，工程上不可能 1-2 周上线。

**修复**：把 V1 收窄成"英语 + 悬疑叙事 niche + 5 个旗舰音色 + Inworld + Kokoro 兜底 + 粘贴文本→mp3"，其余明确转 V2+。`prd.md:627` 这个章节是 PRD 全文的命门，不重写就别动代码。

### R2. 没有差异化承诺，对外话术被自我约束阉割

**位置**：`prd.md:341-348`（5.1 对外表述原则）+ `prd.md:649-655`（推荐副标题 "Long-form AI voice workflow for creators"）

5.1 一连五条"不宣称"，把首页文案变成软弱的工具描述。调研结论是 Raphael 式锐利 H1 才能站住分发。当前副标题既不传播也不可索引。

**修复**：保留"不夸大"原则，但补三条**可宣称**的具体承诺并写进 PRD：

1. 按成片分钟数计费（重生不计费）
2. 旗舰音色 12 个月版本锁定
3. Workspace 级发音词典

这三条是真正的护城河叙事，在 PRD `prd.md:289-313`（产品主张）里完全缺席。

### R3. 计费模型口径与差异化叙事未对齐

**位置**：`prd.md:1763-1782`（19.9-19.10 计费边界）+ `specs/billing-usage-semantics.md`

PRD 现在的计费单位是 `billable_seconds`（`prd.md:1780`），但**没有任何地方说明"按成片分钟、重生不计费"是产品对外承诺**。这意味着 specs 里写的可能是"每次重生都计费"，而调研结论的护城河是"重生不计费"。两者不对齐 = 上线后要么承诺塌方要么毛利塌方。

**修复**：在 `billing-usage-semantics.md` 顶部明确 `regen_billable: false`，并在 PRD `19.9` 里写成原则性结论。

### R4. Inworld Founder pricing 单点失败风险未对冲

**位置**：`prd.md:1738-1742`（19.6 额度建议建立在 Founder 价格上）+ `prd.md:1852-1855`（Founder 锁定截止 2026-05-07）

整套 Free/Pro/Ultimate 分钟额度都建立在 Founder pricing 之上，截止日只剩 **2 周**。如果错过窗口，PRD 里所有定价数字立刻作废。这是一个倒计时性的阻塞风险，但 PRD 把它写成了一条普通注释。

**修复**：在 `prd.md` 顶部"待办决策"或风险章节用显式标注：**2026-05-07 前必须完成 Founder pricing 锁定决策**，否则需启动 PRD 价格回调流程（已在 `21.5` 描述但缺执行触发器）。

## 三、🟡 重要建议

### Y1. 9 种 UI 语言定位错配

**位置**：`prd.md:688-707`

PRD 把 9 语 UI 写成"V1 核心路径 9 语界面"，按 UX 完整性投入。Raphael 模式下，9 语应**只翻首页 + 1-2 个工具页作为 SEO 资产**，不翻帮助/Blog/工作台。当前定义会导致翻译/QA/客服成本爆炸而 SEO 收益有限。

### Y2. 缺独立工具页矩阵（分发漏斗真空）

全文搜索 `prd.md` 没有任何"独立工具页"概念。Raphael 的 7 个独立工具页（/uncrop、/editor 等）是它 170 万 MAU 的核心结构。Phase 7 才上 SEO = 上线即裸奔。

**建议**：在 `26 首页信息架构` 平级新增 `27 独立工具页矩阵`，列出 4-6 个候选：`/youtube-script-to-voice`、`/audio-to-srt`、`/voice-comparison`、`/disclosure-generator`。

### Y3. 三层音色库 + 私有克隆在 V1 过载

**位置**：`prd.md:400-513`（三层音色库 + 建库工作流 + 上传/克隆 + 举报下架流）

8 个状态机 + 6 阶段建库流 + 5 阶段举报流，全部在 V1。调研结论：**早期 99% 用户不会用克隆**。这一整章应整体降级为 V2，V1 只保留"5 个平台旗舰音色"硬编码即可。

### Y4. 免登录试用埋得太深

**位置**：`prd.md:641` 只是 V1 范围的第 11 项，且 `specs/guest-trial-identity.md` 主要写身份归并而非首页主漏斗。

调研结论这是最关键杠杆。应升格为**首页主 CTA + Hero 第一行文案约束**，并在 PRD `26.3 Hero` 显式要求"5 秒内出 mp3，下载时再 gate"。

### Y5. 自部署 / IndexTTS2 在 PRD 中权重过高

**位置**：`prd.md:1929-1953`（21.8 IndexTTS2 大段）+ `prd.md:271`（资产路径强引用）

调研已经明确结论：**Stage 1 不要自部署（除 Kokoro 兜底）**。但 PRD 把 IndexTTS2 写成"现有重要战略资产"占整整一节，会持续诱导工程团队投入预研时间。建议降级为附录或 V3 预研项。

### Y6. 运营后台 9 视图 V1 过重

**位置**：`prd.md:1364-1376`（16.2 九个核心视图）

V1 单用户 + 5 音色 + 单 niche，根本不需要"风控视图、质量评测看板、自动化任务视图"。建议 V1 只保留：用户 / 订阅 / 项目运行 / 克隆四视图，其余进 V2。

### Y7. 数据库 10 张表 vs MVP 实际需求

**位置**：`prd.md:1257-1270`

V1 真实需求：users、projects、project_runs、usage_ledger 四张就够。voice_assets / voice_tags / user_voice_clones 都跟随 R3/Y3 一起进 V2。

### Y8. SRT 技术路径未做实测前不应写死

**位置**：`prd.md:584-625`（7.4-7.6 Google STT + WhisperX）

PRD 把 Google STT + WhisperX 写成"主测试方案"，但没有任何已完成的 benchmark 数据。在 SRT 是核心交付物（`prd.md:561`）的前提下，未实测就写死技术路径是工程风险。建议在 plans 里前置一个 1 周的 SRT spike。

### Y9. 风控规则只到表格，缺触发后果指标

**位置**：`prd.md:2129-2139`

表格只写"升级动作"，没写**误杀率上限**和**对真实用户的影响兜底**。免登录免费 + 强风控的组合最容易把潜在付费用户挡在门外。

## 五、结构性观察（不计分级）

PRD 的真源拆分模式（index + 8 specs）**结构正确但内容超载**。这套治理框架本来是为多 agent 并行和长生命周期产品设计的，套在一个还没上线的 MVP 上 = 治理成本 > 产品价值。建议 V1 阶段把 8 份 specs **合并回主 PRD 作为附录**，等真正进入多人 / 多 agent 协作再拆出。否则后续每改一个规则都要同步 3 个文件（spec → index → PRD 摘要），**摩擦力会让 PRD 在两周内开始腐化**。

---

## 六、总体结论

**🔴 打回，需有条件重写后再放行。**

理由：文档质量优秀，但 V1 范围与已经摸清的市场打法严重脱节（R1）、没有可传播的差异化承诺（R2）、计费护城河未落到 spec（R3）、Founder pricing 倒计时已剩 2 周（R4）。这四点不解决，工程团队即便严格按 PRD 实现，也会做出一个"功能完整但分发为零、价格无法兑现、毛利模型立刻塌方"的产品。

**最小必要修复（按优先级）**：

1. **本周内**重写 `prd.md:627` V1 范围，按"英语 + 悬疑 + 5 旗舰音色 + 免登录"切到最小可上线切片
2. **本周内**在 PRD `5.1` 加入三条可宣称承诺（成片分钟计费 / 12 月音色锁定 / workspace 词典）
3. **2026-05-07 前**完成 Founder pricing 决策，否则启动 `21.5` 价格回调
4. 把三层音色库、9 视图后台、9 语 UI、IndexTTS2 自部署整体降级到 V2 路线图

完成上述四条后可放行进入 plans / 工程实施。
